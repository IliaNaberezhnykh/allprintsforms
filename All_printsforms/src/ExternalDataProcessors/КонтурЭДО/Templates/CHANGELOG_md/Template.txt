## 4.45.7

Доработали наименование документа отгрузки для строки 5а в ПФ 1137. Наименование выводится по следующим правилам:

- Товарная накладная - если в документе реализации есть товары.
- Акт - если в документе реализации только услуги/работы.
- Для УПД  в формате 820 приказа - заполняется "Универсальный передаточный документ".
- Для УПД  в формате 970 приказа - визуализируется как   "тот же", номер и дата не указываются.

## 1.0.0-       Изменения функциональности

  Внесены серьезные изменения в формат хранения данных. Обязательно ознакомьтесь с технической
 информацией ниже  
  Добавлена возможность подписывать исходящие документы облачными сертификатами  
  Отправленные документы в списке сортируются и отбираются по дате отправки, а не дате документа.  
  Добавлена возможность "ручного" обновления данных контрагента (реквизиты, подразделения) с сервера  
  Добавлена возможность загружать полную историю статусов по каждому документу.  
  Добавлена возможность отправлять корректировочный счет-фактуру в формате УКД.  
  Проведена оптимизация быстродействия типовых подключаемых модулей.  
  В управляемой форме добавлена кнопка "Изменить форму" для настройки колонок основного списка.  
  При запуске модуля при авторизации по сертификату автоматически начинается подписание квитанций по
 регламентированному документообороту.  
  Исправлена ошибка: невозможно было запросить уточнение по исправительному УПД.- Техническая
 информация:  
  Настроен контроль совместимости подключаемого модуля и основного модуля. В подключаемом модуле
 должна располагаться экспортная функция "ЭДО_ВерсияAPIПодключаемогоМодуля()", возвращающая ту
 версию API, для которой данный подключаемый модуль был разработан. Это позволит избежать
 некорректной работы подключаемого модуля в случае критичных изменений на стороне основного модуля
 или хранения данных.  
  При данном обновлении необходимо провести ревизию кода в подключаемом модуле и при необходимости
 доработать его под новый формат хранения данных. Особенно это касается работы со входящими
 документами.  
  Добавлено поле "ЗаблокированныйПакет" в справочник "Диадок_ПакетыДокументов"  
  Добавлены поля "ДополнительныйСтатус", "DepartmentId" и "ФункцияДокумента" в справочник
 "Диадок_Документы"  
  Входящие документы теперь всегда объединяются в пакеты (под каждый пакет создается элемент
 справочника "Диадок_ПакетыДокументов").  
  Направление "Inbound" во входящих документах заменено на "Входящее".  
  Перенесены в основную таблицу справочника "Диадок_Документы" следующие дополнительные реквизиты,
 использовавшиеся ранее в обработках для торговых сетей: "ОшибкиПроверкиТекст, ОтветХранилище,
 ДействиеВ1С, ДействиеВДД, АнализПроведен".  
  Добавлен регистр сведений "Диадок_СтатусыДокументов" для хранения полной истории статусов каждого
 документа.
